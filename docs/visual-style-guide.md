## 1. 设计原则

### 1.1 UI 是框架，不是主角

Seed Vault 的内容是 Minecraft 世界截图。截图本身已经充满色彩——草绿、水蓝、暮色橙。UI 的角色是**为截图提供一个安静、可信的框架**，而不是与截图争夺注意力。

这意味着：UI 越中性，截图越突出。

### 1.2 工具感，不是营销感

这个网站的功能是存、查、取种子。类比现实世界：种子库的走廊是混凝土的，不是霓虹灯的。网站应该在视觉上传达同样的东西——可靠、持久、不喧哗。

具体表现：
- 优先使用结构表达层级（间距、对齐、粗细），而非颜色
- 强调色仅用于**一个**动作——"复制种子值"。其他所有交互用中性色
- 没有装饰性元素（渐变背景、发光边框、装饰图标）

### 1.3 颜色不是免费信息

每增加一种颜色，用户就多学一个颜色编码。颜色应该携带可预测的语义：

| 颜色信号 | 含义 | 使用范围 |
|---------|------|---------|
| 冷蓝（primary） | 交互元素、导航 | 链接、按钮、选中态、复制种子值 |
| 绿（Java badge） | 版本端标识 | Java 版徽章 |
| 琥珀（Bedrock badge） | 版本端标识 | 基岩版徽章 |
| 中性灰（neutral） | 结构、分割、次级信息 | 90% 的 UI |

除此之外，不引入其他颜色。无独立的强调色（accent）——Bedrock 徽章的琥珀色仅用于该徽章，不作为全局强调色复用。

## 2. 颜色系统

### 2.1 中性色（Neutral）

UI 的骨干。90% 的界面由中性色构成。

| Token | 色值 | 用途 |
|-------|------|------|
| `neutral-0` | `#fafbfc` | 页面底色（亮色模式） |
| `neutral-50` | `#f0f2f5` | 卡片底色、输入框底色 |
| `neutral-100` | `#e4e7eb` | 分割线、边框 |
| `neutral-200` | `#cbd0d6` | 强边框、聚焦环 |
| `neutral-300` | `#9ca3af` | 禁用文字、placeholder |
| `neutral-400` | `#6b7280` | 次级文字（描述、元信息） |
| `neutral-500` | `#4b5563` | 正文文字 |
| `neutral-600` | `#374151` | 标题文字 |
| `neutral-700` | `#1f2937` | 大标题、强调文字 |
| `neutral-800` | `#111827` | 最深色（亮色模式下极少使用） |
| `neutral-900` | `#0a0e14` | 暗色模式页面底色 |

### 2.2 主色调（Primary）— 冷蓝

用于链接、普通按钮、导航选中态、聚焦环。不是装饰色，是功能色。

| Token | 色值 | 用途 |
|-------|------|------|
| `primary-50` | `#eff6ff` | 极浅底色（选中行、悬停卡片） |
| `primary-100` | `#dbeafe` | 浅底色（标签、badge 底色） |
| `primary-200` | `#bfdbfe` | — |
| `primary-500` | `#3b82f6` | 主按钮、链接、选中态 |
| `primary-600` | `#2563eb` | 主按钮悬停 |
| `primary-700` | `#1d4ed8` | 主按钮按下 |

**使用约束**：`primary-500` 不应出现在非交互元素上。如果一段文字只是标题、不触发任何操作，它就是 neutral 色，不是 primary 色。

### 2.3 版本端徽章色

| Token | 色值 | 用途 |
|-------|------|------|
| `java-green` | `#22c55e` | Java 版徽章文字色 |
| `java-green-bg` | `#f0fdf4` | Java 版徽章底色（亮色模式） |
| `bedrock-amber` | `#d97706` | 基岩版徽章文字色 |
| `bedrock-amber-bg` | `#fffbeb` | 基岩版徽章底色（亮色模式） |

**选色理由**：见 `interaction-design.md` §2.5。

### 2.4 语义色

用于反馈态（成功/错误/警告提示），不用于品牌或装饰。

| Token | 色值 | 用途 |
|-------|------|------|
| `success` | `#16a34a` | Toast 成功、"已验证"徽章 |
| `error` | `#dc2626` | 表单校验错误、删除确认 |
| `warning` | `#d97706` | 兼容性横幅、警告提示 |

### 2.5 颜色使用率预算

一个健康的 Seed Vault 页面，颜色面积分布约：

```
neutral（灰白系）  ████████████████████████████████  85%
primary（冷蓝）     ████                             10%
semantic（绿/红）   █                                2%
edition badge      ██                               3%
```

如果 primary 超过 12%，说明颜色正在被滥用。重新审查——哪些蓝色元素可以改成灰色。

## 3. 字体层级

### 3.1 字体栈

```css
/* 正文 */
font-family: -apple-system, BlinkMacSystemFont, "Segoe UI", Roboto,
             "PingFang SC", "Microsoft YaHei", "Helvetica Neue", sans-serif;

/* 种子值、坐标 */
font-family: "JetBrains Mono", "Cascadia Code", "Fira Code",
             "SF Mono", "Consolas", "Menlo", monospace;
```

**决策**：不引入外部字体服务。系统字体栈在中文和英文环境下质量都足够好（PingFang SC 在 macOS/iOS 上是第一方的中文字体），等宽字体优先 JetBrains Mono 但也提供多个 fallback。对 MVP 来说零额外加载是更重要的。

### 3.2 字号 / 字重 / 行高表

所有尺寸基于 16px 根字号（`1rem = 16px`）。

| 层级 | 字号 | 字重 | 行高 | 用途 |
|------|------|------|------|------|
| `text-xs` | 0.75rem (12px) | 400 | 1.5 | 标签、徽章、次要元信息 |
| `text-sm` | 0.875rem (14px) | 400 | 1.5 | 描述文字、列表项、筛选器选项 |
| `text-base` | 1rem (16px) | 400 | 1.6 | 正文、输入框、按钮 |
| `text-lg` | 1.125rem (18px) | 500 | 1.5 | 卡片标题、子标题 |
| `text-xl` | 1.25rem (20px) | 600 | 1.4 | 详情页标题、区块标题 |
| `text-2xl` | 1.5rem (24px) | 600 | 1.3 | 页面标题 |
| `text-3xl` | 1.875rem (30px) | 700 | 1.2 | 首页品牌标题（"SEED VAULT"） |

### 3.3 种子值专用

种子值是全站最重要的文字，它在视觉上必须与所有其他文字区分开：

```
字号:   1.5rem (24px)         ← 比正文大两级
字重:   500                   ← 中等，不是粗体（粗体在等宽字体下可读性下降）
字体:   JetBrains Mono       ← 等宽，暗示"这是数据/代码"
letter-spacing: 0.02em       ← 微增字间距，数字之间不粘连
颜色:   neutral-700           ← 深色，不是 primary 蓝（值不是链接）
```

### 3.4 中文字体注意事项

- PingFang SC（macOS/iOS）在 14px 及以上表现优秀，12px 及以下略模糊 → 中文标签最小字号 13px（0.8125rem）
- Microsoft YaHei（Windows）在 dark mode 下字形偏细，应适当提高字重（400→500）
- 种子标题、描述是用户生成内容，可能纯中文、纯英文、中英混排 → `text-base` 的 1.6 行高在各种混排下都可读

## 4. 间距与网格

### 4.1 基础单元

基于 4px 网格。所有间距、内边距、尺寸都是 4 的倍数。

```
0   4   8   12  16  20  24  32  40  48  64  80  96
```

### 4.2 语义 Token

| Token | 值 | 用途 |
|-------|---|------|
| `space-1` | 4px | 标签组内间距、图标与文字间距 |
| `space-2` | 8px | 卡片内元素间微间距 |
| `space-3` | 12px | 同一信息组内间距 |
| `space-4` | 16px | 卡片内边距、表单字段间距 |
| `space-5` | 20px | — |
| `space-6` | 24px | 区块内边距、段落间距 |
| `space-8` | 32px | 卡片网格间距、侧边栏内边距 |
| `space-10` | 40px | 页面区块间距 |
| `space-12` | 48px | — |
| `space-16` | 64px | 首页品牌区与内容区间距 |
| `space-20` | 80px | 页面首尾留白 |

### 4.3 容器最大宽度

```
内容区最大宽度:      1152px (72rem)
种子详情页主内容区:    min(65%, 720px)
种子详情页侧边栏:      min(35%, 384px)
筛选器面板宽度:        256px
种子卡片最小宽度:      256px
```

### 4.4 卡片网格

使用 CSS Grid `auto-fill` + `minmax`：

```css
.seed-grid {
  display: grid;
  grid-template-columns: repeat(auto-fill, minmax(272px, 1fr));
  gap: 24px;        /* space-6 */
}
```

- `minmax(272px, 1fr)` 确保卡片最小宽度舒适（272 = 256 内容 + 16 两侧 padding），且自动填充
- 断点 640px 以下改为单列 `grid-template-columns: 1fr`

## 5. 圆角与边框

### 5.1 圆角

**规则**：圆角用于软化矩形，不是装饰手段。克制且系统化。

| Token | 值 | 用途 |
|-------|---|------|
| `radius-sm` | 4px | 输入框、按钮、标签、徽章 |
| `radius-md` | 8px | 卡片、弹窗、筛选面板 |
| `radius-full` | 9999px | 版本徽章（pill 形状） |

- 卡片图片区顶部保留 `radius-md` 圆角（与卡片一致），底部直角
- 不做圆形头像（用户头像使用 `radius-sm`）

### 5.2 边框

| Token | 色值 | 宽度 | 用途 |
|-------|------|------|------|
| `border-subtle` | `neutral-100` | 1px | 卡片边框、输入框边框、分割线 |
| `border-default` | `neutral-200` | 1px | 聚焦框、下拉菜单边框 |
| `border-focus` | `primary-500` | 2px | 输入框聚焦环 |

- 不使用 `box-shadow` 替代 `border` 做聚焦环。2px 实线边框是最可靠、最可访问的方案

## 6. 阴影与深度

### 6.1 原则

阴影表达的是**功能层级**，不是视觉效果。如果元素需要浮在另一个元素上方（弹窗、下拉、悬停卡片），它才需要阴影。不需要的时候就没有阴影。

### 6.2 阴影 Token

| Token | 值 | 用途 |
|-------|---|------|
| `shadow-card` | `0 1px 3px rgba(0,0,0,0.06)` | 卡片悬浮态（浏览页 hover） |
| `shadow-dropdown` | `0 4px 12px rgba(0,0,0,0.08)` | 下拉菜单、弹出面板 |
| `shadow-modal` | `0 8px 32px rgba(0,0,0,0.12)` | 弹窗、灯箱 |

### 6.3 禁止

- ❌ 彩色阴影（`rgba(59,130,246,0.3)` 等）
- ❌ 多层阴影制造"发光"效果
- ❌ 阴影动画（从无阴影过渡到有阴影除外——这是功能性的悬停反馈）
- ❌ 卡片默认带阴影。卡片默认用边框（`border-subtle`），仅在 hover 时加 `shadow-card`

## 7. 组件标记

### 7.1 按钮

三种层级，语义互斥。

```
┌──────────────────┐  ┌──────────────────┐  ┌──────────────────┐
│   主要操作         │  │   次要操作         │  │   危险操作         │
│   primary-500     │  │   neutral-0       │  │   neutral-0       │
│   白色文字         │  │   neutral-600 文字  │  │   error 文字      │
│   无边框           │  │   neutral-200 边框  │  │   error 边框      │
└──────────────────┘  └──────────────────┘  └──────────────────┘
```

**"复制种子值"按钮**使用 primary（冷蓝）样式，与主要操作按钮一致：

```
┌──────────────────────────────┐
│   📋 复制种子值               │  primary-500 底色
│                              │  白色文字
│   hover: primary-600         │  radius-sm, 无边框
│   active: primary-700        │
└──────────────────────────────┘
```

这是全站最显眼的按钮——在详情页侧边栏中占满宽度（`width: 100%`），视觉层级与其他页面的 primary 按钮相同，不额外引入强调色。

按钮最小触摸目标：40px 高度（桌面端），48px（移动端）。

### 7.2 输入框

```
┌──────────────────────────────────────────┐
│  placeholder 文字 (neutral-300)           │  neutral-50 底色
│                                          │  border-subtle 边框
└──────────────────────────────────────────┘  radius-sm

聚焦态：
┌══════════════════════════════════════════┐
│  用户正在输入...                          │  border-focus (primary-500, 2px)
│                                          │  底色不变
└══════════════════════════════════════════┘
```

- 聚焦时边框色从 `border-subtle` 过渡到 `border-focus`，transition 150ms
- 不使用 `box-shadow` 做发光聚焦
- 禁用态：`neutral-100` 底色 + `neutral-300` 文字

### 7.3 种子卡片

```
┌──────────────────────────┐
│                          │
│   截图 16:9              │  上部，撑满宽度
│                          │
├──────────────────────────┤  ← 分割由截图底部 + padding 自然形成
│                          │
│  种子标题                 │  text-lg, weight 500, neutral-700
│  描述文字（2行截断）       │  text-sm, neutral-400
│                          │
│  [Java 1.21.4] 🏕️生存   │  版本徽章 + 标签
│                          │
│  👤投稿者  ❤️42  👁1.2k  │  text-xs, neutral-400
│                          │
└──────────────────────────┘

默认态:  border-subtle 边框, 无阴影, radius-md
Hover:   shadow-card, 边框色过渡到 neutral-200
```

- 卡片 hover 时增加"复制"快捷按钮（详见 `interaction-design.md` §10.2）
- 卡片内部不嵌套圆角（截图顶部跟随卡片圆角，无需单独设置）

### 7.4 版本徽章

```
Java:  ┌───────────────┐    Bedrock: ┌───────────────┐
       │ Java · 1.21.4 │             │ 基岩 · 1.21.60 │
       └───────────────┘             └───────────────┘
```

| 属性 | Java | Bedrock |
|------|------|---------|
| 文字色 | `java-green` | `bedrock-amber` |
| 底色 | `java-green-bg` | `bedrock-amber-bg` |
| 形状 | `radius-full` pill | `radius-full` pill |
| 字号 | `text-xs` (12px) | `text-xs` (12px) |
| 字重 | 500 | 500 |
| 内边距 | 2px 10px | 2px 10px |

### 7.5 标签（Tag / Chip）

```
┌──────────┐  ┌──────────┐  ┌──────────┐
│ 🏕️ 生存  │  │ 🍄 稀有  │  │ 🏘️ 村庄  │
└──────────┘  └──────────┘  └──────────┘
```

- 底色：`neutral-100`
- 文字：`neutral-600`，`text-xs`，400
- 形状：`radius-sm`
- 选中（筛选器内）：底色 `primary-100`，文字 `primary-700`
- 可交互标签 hover：底色加深至 `neutral-200`

### 7.6 Toast / 通知

```
┌─────────────────────────────────────────┐
│  ✅ 种子值已复制到剪贴板         [✕]     │
└─────────────────────────────────────────┘
```

- 底色：`neutral-800`（无论亮暗模式）
- 文字：白色，`text-sm`
- 位置：页面底部居中，距底部 24px
- 出现/消失：fade + slide-up，200ms
- 自动消失：2.5s

### 7.7 骨架屏（Skeleton）

```
┌──────────────────────┐
│ ████████████████████ │  ← 截图占位，neutral-100
│ ████████████████████ │
├──────────────────────┤
│ ████████████         │  ← 标题占位，neutral-100
│ ██████████████       │  ← 描述占位
│ ████                 │  ← 徽章占位
└──────────────────────┘
```

- 骨架块使用 `neutral-100` 底色
- 脉冲动画：`opacity 1 → 0.5 → 1`，周期 1.5s，ease-in-out
- 不使用渐变动画（"shimmer"效果），纯色脉冲更安静

### 7.8 Microsoft 登录按钮

遵循 Microsoft 官方品牌准则。使用 Microsoft 提供的官方 SVG Logo 资源（不可修改、不可自绘）。

```
┌─────────────────────────────────┐
│  [Microsoft Logo SVG]           │  底色: #FFFFFF（或跟随项目 neutral-0）
│  Sign in with Microsoft         │  文字: #5E5E5E, Segoe UI 15px, weight 600
│                                 │  边框: 1px solid #8C8C8C
└─────────────────────────────────┘  radius-sm
```

| 属性 | 值 |
|------|---|
| 字体 | 项目系统字体栈（Segoe UI / San Francisco / PingFang SC）15px, weight 600 |
| 文字色 | `#5E5E5E` |
| 底色 | 跟随页面背景（`neutral-0`），保持与其他按钮一致的视觉环境 |
| 边框 | `1px solid #8C8C8C` |
| Logo | Microsoft 官方 SVG（可从 Microsoft 品牌门户下载） |
| 最小宽度 | 240px（确保文字和 Logo 不拥挤） |
| 高度 | 44px |

**暗色模式**：文字色切换为 `#FFFFFF`，底色切换为 `#2F2F2F`（使用 Microsoft 暗色主题规格），去除边框。

**设计决策**：字体栈与项目系统字体保持一致而非强制 Segoe UI（Microsoft Q&A 中对此有宽容——"using a single system font across all auth provider buttons is acceptable"）。

### 7.9 Minecraft 皮肤头像

头像来自 `mc-heads.net/avatar/{uuid}/{size}`。这是像素艺术（8×8 面部），必须保持方形或小圆角，不能裁切成圆形。

```
┌──────┐     ┌──────┐     ┌──────────┐
│      │     │      │     │          │
│  🧑  │     │  🧑  │     │    🧑    │
│      │     │      │     │          │
└──────┘     └──────┘     └──────────┘
 24px         32px          80px
卡片投稿者    导航栏        用户主页
```

| 场景 | 尺寸 | 圆角 | 来源 |
|------|------|------|------|
| 卡片投稿者 | 24px | `radius-sm` (4px) | `mc-heads.net/avatar/{uuid}/24` |
| 导航栏 | 32px | `radius-sm` (4px) | `mc-heads.net/avatar/{uuid}/32` |

## 8. 动效原则

### 8.1 何时需要动画

动画只用于三种情况：
1. **反馈**：用户操作后确认系统接收到了（按钮按下、复制成功）
2. **过渡**：状态变化需要被注意到但不应突兀（弹窗打开、页面切换）
3. **系统状态**：告知等待（骨架屏脉冲）

除此之外不添加动画。特别地：
- ❌ 不做入场动画（scroll-triggered reveal、fade-in-on-scroll）
- ❌ 不做装饰性悬停动画（scale、rotate、color shift）
- ❌ 不做 stagger / sequenced animations

### 8.2 时长与缓动

| 类型 | 时长 | 缓动 | 用途 |
|------|------|------|------|
| 微交互 | 150ms | `ease-out` | 按钮 click 反馈、hover 颜色过渡 |
| 标准过渡 | 200ms | `ease-out` | 弹窗开/关、模态出现 |
| 通知 | 200ms | `ease-out` | Toast 入场/退场 |

**不使用 `ease-in-out`**（对称缓动让响应感迟钝）。出现用 `ease-out`（快速出现），消失用 `ease-in`（快速离开后减速留给用户反应时间）。

### 8.3 Hover 过渡

所有 hover 状态使用统一的过渡：

```css
transition: background-color 150ms ease-out,
            border-color 150ms ease-out,
            box-shadow 150ms ease-out;
```

- 不过渡 `color`（文字色变化足够快，不需要动画）
- 不过渡 `transform`（hover 不做缩放/位移）
- 不过渡 `opacity`（除非是元素出现/消失）

## 9. 图标系统

### 9.1 两类图标，两种策略

| 类别 | 方案 | 理由 |
|------|------|------|
| **功能性图标**（按钮、导航、输入框、状态指示） | Lucide SVG | 跨平台一致的线条风格，视觉重量轻 |
| **标签图标**（数据库 `tags.icon` 字段） | Emoji（VARCHAR(8) 存储） | 标签图标是**内容数据**而非 UI chrome，emoji 作为标签的视觉分类符是合适的；用户生成内容中使用 emoji 是 Web 惯例 |

功能性图标绝不可使用 emoji（不同平台渲染差异大）。标签图标可继续使用 emoji（它们是数据，且在标签 chip 中作为装饰性前缀，不承载独立信息）。

### 9.2 图标集

使用 **Lucide**（`lucide-vue-next`）。选择理由：
- 线条风格，视觉重量轻，不抢占注意力
- 与系统字体搭配自然（基于 24px 网格的 1.5px 描边）
- Tree-shakable，按需加载
- MIT 许可，无版权顾虑

## 10. 暗色模式

### 10.1 原则

暗色模式是**功能**（游戏中查看种子网站不刺眼），不是**美学**。暗色模式的颜色映射应保持所有语义、层级和信息密度不变。

### 10.2 中性色映射

| 亮色 Token | 暗色色值 | 说明 |
|-----------|---------|------|
| `neutral-0` (#fafbfc) | `#111827` | 页面底色 |
| `neutral-50` (#f0f2f5) | `#1f2937` | 卡片/输入框底色 |
| `neutral-100` (#e4e7eb) | `#374151` | 边框 |
| `neutral-500` (#4b5563) | `#9ca3af` | 正文文字 |
| `neutral-600` (#374151) | `#d1d5db` | 标题文字 |
| `neutral-700` (#1f2937) | `#f3f4f6` | 强调文字 |

### 10.3 暗色模式特有规则

- 卡片默认带 1px `neutral-700` 边框（暗色下默认无边框的卡片会融入背景）
- 阴影在暗色下增大透明度偏移（`rgba(0,0,0,0.12)` → `rgba(0,0,0,0.30)` 等效，因为暗色背景下阴影天生更不明显）
- Java 版徽章底色：`#052e16`（深绿）；Bedrock 徽章底色：`#451a03`（深琥珀）
- "正版玩家"徽章底色：`#052e16`（同 Java 徽章深色底色）
- Microsoft 登录按钮：文字 `#FFFFFF`，底色 `#2F2F2F`，无边框（使用 Microsoft 暗色主题规格）
- mc-heads.net 头像 URL 不变（该服务返回的图片本身在暗色背景下自然显示良好，皮肤头像不做透明度处理）
- Toast 底色保持 `neutral-800`（在暗色模式下不变，因为已足够深）

### 10.4 模式切换

- 默认跟随 `prefers-color-scheme`
- 手动切换存入 `localStorage`，key: `seedvault_theme`，value: `light` | `dark` | `auto`
- 切换时不触发页面重排（CSS 变量切换，在 `:root` 和 `[data-theme="dark"]` 中定义完整）
- 不推荐使用 `class` 切换 → 使用 `data-theme` 属性，语义更明确

## 11. 禁止清单

这些是被明确禁止的视觉模式。它们中的每一个都单独出现于某些"现代 UI"中，组合起来即构成所谓的 vibe coding 美学。

### 11.1 颜色

- ❌ 渐变背景（包括但不限于 `linear-gradient` 做 section 背景、按钮背景、文字色）
- ❌ 蓝色-紫色渐变组合
- ❌ 高饱和强调色（`#FF4500`, `#FF0000`, `#00FF00` 等未经调灰的纯色）
- ❌ 彩色文字作为装饰（标题渐变色、彩色标签随机色）
- ❌ `backdrop-filter: blur()` 玻璃态效果

### 11.2 边框与阴影

- ❌ 彩色阴影 / 发光（`box-shadow: 0 0 20px rgba(59,130,246,0.5)`）
- ❌ 渐变边框
- ❌ 装饰性动画边框（旋转渐变边框、脉冲边框）
- ❌ 巨大圆角（`border-radius > 12px` 除了 pill 徽章）

### 11.3 动效

- ❌ 入场滚动动画（`IntersectionObserver` 驱动的 fade-in、slide-up）
- ❌ 装饰性悬停缩放（`transform: scale(1.05)`）
- ❌ 页面切换动画（SPA 路由切换即可，不需要额外过渡）
- ❌ 浮动/漂移动画（除非是骨架屏的脉冲——那是功能性反馈）

### 11.4 字体

- ❌ 引入第三方字体服务（Google Fonts 等）
- ❌ 装饰性字体用于 UI（手写体、花体、显示字体）
- ❌ 字重 800+ 的粗体（系统字体在 700 以上质量控制参差不齐）

### 11.5 布局

- ❌ 全屏 Hero 区域（首页品牌区不超过视口高度的 40%）
- ❌ 固定宽高比视频背景
- ❌ 装饰性波浪 / SVG 分隔符
- ❌ 卡片倾斜 / 3D 旋转效果
